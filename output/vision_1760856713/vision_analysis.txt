[Detailed technical specification of the visual design]

1) Layout structure
- Page type: Single-screen landing page with a centered hero and a 3-column footer.
- Overall regions (semantic HTML): 
  - header: brand name “BLAKE Inc.”
  - main: hero with email capture form and hero illustration
  - footer: three columns labeled “Disclaimer”, “Email”, “Social Media”
- Container and alignment:
  - max-width: 1200px (lg), 960px (md), full-width with side padding (sm)
  - horizontal padding: 24px (sm), 32px (md+)
  - All hero content center-aligned on the page.
- Visual “frame” (the thick border drawn around the page):
  - Implement as an inset frame around the viewport: body { box-shadow: inset 0 0 0 16px var(--ink); border-radius: 12px; } 
  - Provide 16px inner safe padding so content doesn’t touch the frame.
- Grid/flex structure:
  - Header: single row (center).
  - Hero: vertical stack using flex-column; the email form is an inline row using flex-row.
  - Footer: 3-column grid (1fr 1fr 1fr) on md+; single-column stack on small screens.
- Spacing rhythm (vertical):
  - Header → hero: 24–32px
  - Title → form: 16–24px
  - Form → hero illustration: 24–40px
  - Main → footer: 48px
  - Footer row items: gap 24px (md+), 16px (sm)

2) UI components

A) Header
- Components: 
  - Logo/brand wordmark “BLAKE Inc.” (clickable to home)
- States: default; hover/focus adds subtle underline or opacity change.

B) Hero section
- Components:
  - H1: “BLAKE Inc.”
  - Email capture form (inline):
    - Text input: type=email, placeholder “Email address”
    - Primary button: “Sign Up”
  - Hero illustration: circular smiley (use an SVG or image; centered)
- Hierarchy/nesting:
  - <section role="region" aria-labelledby="hero-title">
    - <h1 id="hero-title">BLAKE Inc.</h1>
    - <form aria-label="Join mailing list">
      - <label for="email">Email address</label>
      - <input id="email" type="email">
      - <button type="submit">Sign Up</button>
    - <figure aria-label="Friendly smiley illustration">…</figure>
- Component states:
  - Input: default, focus (2px focus ring), error (red border + helper text), valid (neutral), disabled.
  - Button: default, hover (elevate/brighten), focus-visible (outline), active (pressed), disabled (reduced contrast), loading (spinner, aria-busy).
  - Form: success state shows inline confirmation “Thanks—check your email.”

C) Footer (3 columns)
- Left: “Disclaimer” heading + text link to legal/terms
- Center: “Email” heading + a visible mailto link or small contact CTA
- Right: “Social Media” heading + icon buttons (e.g., X/Twitter, LinkedIn, Instagram)
- States:
  - Links/icons: hover color shift; focus rings; active pressed state.
  - Icons should have aria-labels (“Follow on LinkedIn”).

3) Visual design

- Color palette (derived from sketch with suggested accessible system):
  - --bg: #FFFFFF
  - --ink (primary text/frame): #111827
  - --muted-text: #4B5563
  - --primary (CTA): #2563EB
  - --primary-hover: #1D4ED8
  - --focus: #0EA5E9
  - --error: #DC2626
  - Footer background may stay --bg or a subtle tint #F8FAFC.
- Typography:
  - Font family: System stack or Inter; brand wordmark may use a display weight.
  - H1: 48–64px, 700 weight, tight line-height 1.1, centered.
  - Body: 16px, line-height 1.6.
  - Button: 16px, 600 weight.
  - Footer headings: 14–16px, 600; footer links: 14–16px.
- Icons and images:
  - Smiley as a centered 240–320px SVG circle with stroke matching --ink.
  - Social media icons from Lucide, Heroicons, or Feather; 20–24px size.
- Borders, shadows, effects:
  - Page frame: 16px inset “border” via box-shadow.
  - Input: 1px solid #D1D5DB; radius 8px; focus ring 2px #0EA5E9 (offset).
  - Button: solid fill --primary; radius 8px; elevation shadow: 0 2px 6px rgba(0,0,0,0.12); hover increases to 0 4px 10px.
  - Footer columns have no card borders; rely on spacing.

4) Interactions

- Header brand click: navigates to top (“/”).
- Email form:
  - Validation: HTML5 + custom pattern /^\S+@\S+\.\S+$/.
  - If invalid on submit: prevent submit; show aria-live polite error under input (“Enter a valid email”).
  - On success: POST to /api/subscribe (or ESP endpoint). Show success message inline and optionally replace form with “You’re in!” state.
  - Keyboard:
    - Tab order: input → button
    - Enter on input triggers submit.
- Social icons: open in new tab with rel="noopener noreferrer".
- Footer links: navigate to legal/contact pages.
- Animations/transitions:
  - 150–200ms ease transitions for hover/focus on buttons/links.
  - Optional: subtle fade-in of hero illustration on load (200–300ms, prefers-reduced-motion respected).

5) Technical recommendations

- Framework: React with Next.js (SSR/SSG friendly landing page).
- Styling:
  - Tailwind CSS for rapid layout and design tokens, or CSS Modules with design tokens defined as CSS variables.
- Forms:
  - react-hook-form + zod for schema validation; fallback to native validation if non-JS.
  - Honeypot or reCAPTCHA v3 for bot
