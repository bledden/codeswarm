[Detailed technical specification of the visual design]

1) Layout Structure
- Page regions (semantic HTML):
  - header: centered brand name “BLAKE Inc.”
  - main: hero section with email capture form and a large smiley graphic beneath
  - footer: three-column link area (“Disclaimer”, “Email”, “Social Media”)
- Grid/flex:
  - Use a 12-column responsive grid with a centered container
    - container max-width: 1200px (xl), 1024px (lg), 768px (md), 640px (sm), full width on smaller
    - gutters: 24px (desktop), 16px (tablet), 12px (mobile)
  - Header: single centered row, items centered horizontally and vertically
  - Hero: vertical stack; form centered horizontally; graphic centered below form
  - Footer: 3 columns on md+ (each 4/12), stacked vertically on small screens
- Spacing and alignment (8pt scale):
  - header height: 72px; top/bottom padding 16px
  - hero padding-top: 64px (mobile 40px), padding-bottom: 64px
  - space between H1 and form: 24px
  - space between form and smiley graphic: 32–48px
  - footer padding: 32px top/bottom
- Optional “frame” treatment (to echo the sketch border):
  - page wrapper with 12px solid border #111 and 16px radius; inner padding 16px; use only if desired

2) UI Components
- Header
  - BrandTitle: text logo “BLAKE Inc.”
- Hero
  - EmailSignupForm (form element)
    - TextInput: type=email, placeholder=“you@example.com”, label=“Email”
    - PrimaryButton: “Sign Up”
    - Inline validation message area below input
  - HeroGraphic: decorative smiley (SVG)
- Footer
  - FooterColumn (x3)
    - Left: “Disclaimer” (link)
    - Center: “Email” (mailto link)
    - Right: “Social Media” (row of icon links or a single link label)
- Component hierarchy
  - App
    - Header(BrandTitle)
    - Main(Hero(EmailSignupForm(TextInput, PrimaryButton, Message), HeroGraphic))
    - Footer(FooterColumn, FooterColumn, FooterColumn)
- Component states
  - TextInput
    - default: light border
    - focus: high-contrast outline/ring
    - valid: subtle success border (optional)
    - error: red border, message text visible
    - disabled: reduced opacity, no shadow
  - PrimaryButton
    - default: solid primary color
    - hover: slightly darker background
    - active: pressed state (translateY(1px) or darker)
    - focus-visible: 2px outline/focus ring
    - disabled: 60% opacity, no hover
  - Links (footer)
    - hover: underline, color shift
    - focus-visible: outline ring

3) Visual Design
- Color palette (WCAG-friendly)
  - Background: #FFFFFF
  - Text primary: #111827 (gray-900)
  - Text secondary: #4B5563 (gray-600)
  - Border: #E5E7EB (gray-200)
  - Primary/CTA: #2563EB (blue-600)
  - Primary hover: #1D4ED8 (blue-700)
  - Focus ring: #93C5FD (blue-300) at 2–3px
  - Error: #DC2626 (red-600)
  - Optional frame/border: #111111
- Typography
  - Headings: “Inter”, “Montserrat”, or “Poppins”; weight 700
    - H1 “BLAKE Inc.”: 48–56px desktop, 32–36px mobile, letter-spacing slight +0.2px
  - Body: “Inter”/system-ui; size 16px/24px line-height
  - Button label: 16px, weight 600
  - Footer link labels: 14–16px
- Icons and images
  - HeroGraphic: inline SVG smiley (stroke 4px, 280–360px diameter desktop; 200–240px mobile); stroke color: #111827 or primary
  - Social icons (if used): SVGs (Twitter/X, LinkedIn, Instagram); single-color filled to currentColor
- Borders, shadows, effects
  - Input: 1px border #E5E7EB; 8px radius; focus ring 2px outside
  - Button: 8px radius; no shadow or subtle shadow: 0 1px 2px rgba(0,0,0,0.06)
  - Optional group styling: input and button as a compound control on desktop (shared 8px radius; input right corners 0, button left corners 0)

4) Interactions
- Email signup flow
  - Submit on click of “Sign Up” or Enter key
  - Validation: required, email format (RFC 5322 basic), max length 254
  - Error states: show inline red text beneath input; announce via aria-live="polite"
  - Success: replace form with confirmation message “Thanks! Please check your inbox.” or show inline success banner
  - Optional: POST /api/subscribe { email } with CSRF protection; handle loading state (spinner on button, disable controls)
- Navigation
  - “Disclaimer” links to /disclaimer
  - “Email” uses mailto:support@blake.inc
  - “Social Media” links open in new tab with rel="noopener noreferrer"
- Animations/transitions (reduced-motion friendly)
  - 150–200ms ease-in-out transitions for color, background, border-color
  - Optional subtle fade-in for hero graphic on page load
  - Respect prefers-reduced-motion: reduce/disable animations

5) Technical Recommendations
- Framework: Next.js (React) for fast static generation and simple API routes
- Styling: Tailwind CSS for rapid iteration; alternatively CSS Modules or Vanilla Extract
- Forms/validation: React Hook Form + Zod (client) with server-side validation in API route
- Icons: Hero SVG inline; social icons via Heroicons or Simple Icons
- Analytics/Email
  - Email service: Resend/Mailchimp
