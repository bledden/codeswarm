Grid/flex structure
- Container: max-width 1100–1200px; horizontal padding 16–24px; centered.

4) Interactions

Primary flow
- User enters email → clicks “Sign Up” or presses Enter.
- Form validates email (HTML5 + custom regex) and calls subscription endpoint.
- On success: replace CTA group with inline confirmation message (“Thanks! Check your email.”) and a small checkmark icon. Announce via aria-live.
- On error: show inline error message under input; keep focus on input; announce via aria-live.

Element behaviors
- Header brand: clicking returns to homepage.
- Footer links:
  - Disclaimer → open /disclaimer in same tab.
  - Email → opens mailto: or a “Contact” page; prefer contact form page for accessibility.
  - Social Media → opens a page with social icons or direct external links (target=_blank with rel="noopener").

Keyboard and focus
- Tab order: Brand → Email input → Sign Up button → Footer links left to right.
- Enter in input submits form.
- Escape in input clears if you implement a clear button.

Animations or transitions
- Button/background color transitions: 150–200ms ease.
- Hero illustration subtle entrance: fade-up 150ms, 8px translateY.
- Focus rings appear instantly (no transition).
